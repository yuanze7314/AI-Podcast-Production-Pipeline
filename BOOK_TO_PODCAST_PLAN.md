# 书籍转AI播客工作流开发计划

## 一、需求概述

开发一个将电子书（PDF/EPUB/MOBI）转换为AI播客的完整工作流，包含：
1. 书籍内容解析与结构化
2. 生成章节大纲和各章节摘要（含原句）
3. 按章节生成双人对话播客稿件
4. 将稿件转换为音频

## 二、技术选型

| 模块 | 技术方案 |
|------|----------|
| 开发语言 | Node.js |
| 电子书解析 | pdf-parse (PDF), epub-parser (EPUB) |
| AI模型接口 | OpenAI/Anthropic/DeepSeek 多模型适配 |
| 音频合成 | edge-tts (微软免费TTS) |
| 配置管理 | dotenv + YAML/JSON |

## 三、工作流架构

```
电子书文件
    ↓
1. 文档解析模块 → 提取纯文本内容
    ↓
2. 大纲生成模块 → AI生成书籍章节树
    ↓
3. 章节摘要模块 → 各章节内容总结（含原句引用）
    ↓
4. 播客稿件模块 → 双人对话格式稿件
    ↓
5. 音频合成模块 → Edge TTS生成双语音频
    ↓
输出：播客音频文件 + 稿件文档
```

### 4.2 核心流程

**阶段1: 文档解析**
- 输入: PDF/EPUB文件
- 输出: 纯文本内容 + 元数据（书名、作者）

**阶段2: 大纲生成**
- AI分析全书，生成层级章节大纲
- 输出: JSON格式的章节树

**阶段3: 章节摘要**
- 遍历每个章节，AI生成摘要
- 要求: 包含核心观点 + 原文引用（2-3处）

**阶段4: 对话稿件**
- 按章节生成双人对话
- 设计两个AI角色（主持人+嘉宾）
- 对话风格: 轻松聊天

**阶段5: 音频合成**
- 使用edge-tts生成双语音频
- 支持选择不同音色
- 按章节拆分输出

## 五、已确认的设计选择

## 七、核心 Prompt 设计

### 大纲生成 Prompt
```
分析以下书籍内容，生成结构化的章节大纲。

要求：
1. 识别主要章节和子章节
2. 输出JSON格式，包含章节标题和简短描述
3. 章节数量控制在5-15个之间

书籍内容：
{{content}}
```

### 章节摘要 Prompt
```
根据以下章节内容，生成摘要。

要求：
1. 总结核心观点（100字以内）
2. 引用2-3处原文原句，用引号标注
3. 保持原文的准确含义

章节标题：{{chapter_title}}
章节内容：{{chapter_content}}
```

### 对话稿件 Prompt
```
将以下章节内容改写为轻松聊天风格的播客对话。

角色设定：
- 主持人（女）：引导话题，提出问题
- 嘉宾（男）：分享观点，解答疑问

风格要求：
- 像朋友聊天一样自然
- 不要太正式，保持轻松氛围
- 适当加入语气词和口语表达
- 总对话时长约3-5分钟

章节摘要：{{summary}}
原文引用：{{quotes}}
```

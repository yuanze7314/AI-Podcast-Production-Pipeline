# 前端生产控制台精简改造设计

日期：2026-06-14

## 背景

当前前端已经能跑通 PDF 解析、章节确认、LangGraph 脚本生成、TTS、合成和导出流程，但主界面把项目列表、章节结构、全文、脚本块、产物、任务历史、调试信息和多类操作同时展示出来。对批量生产一本书而言，信息密度过高，用户很难第一眼判断整本书做到哪一步、哪里卡住、下一步该按哪个按钮。

本次改造的目标是把首页从“全功能罗列面板”调整为“批量生产控制台”。用户已确认采用极简方向：主屏强调整本书生产进度，项目列表放进左上角项目切换抽屉，脚本正文和修改能力移入章节展开页。

## 设计目标

- 主屏只回答三个问题：整本书进度如何、当前批处理卡在哪里、下一步做什么。
- 项目列表保留清晰入口，但不常驻占用左侧空间。
- 章节队列展示阶段和动作，不在主屏展开脚本正文、原文、分析、审核和日志。
- 脚本修改进入独立展开页或抽屉，形成更专注的编辑体验。
- 保持现有 React/Vite 技术栈和后端 API，不引入新的 UI 框架。
- 在改造过程中拆分过大的前端文件，让组件边界更清楚，方便后续迭代。

## 非目标

- 不改变后端数据模型和已完成的 LangGraph API。
- 不新增认证、多人协作、云端部署或权限系统。
- 不重新设计完整品牌体系。
- 不移除现有能力；低频功能会被折叠、迁移或降级展示。

## 信息架构

### 1. 顶部项目区

左上角显示当前项目和书名，并提供项目切换入口。入口可使用图标按钮加当前项目标题区域，点击后打开项目切换抽屉。

顶部右侧保留全局操作：刷新、导入 PDF、继续批量生产。按钮数量保持克制，低频操作进入更多菜单或项目抽屉。

### 2. 项目切换抽屉

项目列表不再作为常驻左侧栏。点击左上角项目入口后，从左侧打开覆盖式抽屉。

抽屉内容包括：

- 项目搜索框，支持按项目名、书名或 PDF 名称查找。
- 当前项目高亮，展示生产进度和章节数量。
- 最近项目列表，展示项目名、状态和更新时间。
- 导入 PDF 与新建项目按钮。
- 关闭按钮和点击遮罩关闭。

选择项目后关闭抽屉，并加载对应项目的章节、脚本、任务和产物数据。

### 3. 批量生产主屏

主屏分为三块。

第一块是整本书完成度卡片，展示总进度、脚本完成数、确认数、音频数、错误数和预计剩余时间。数字使用等宽或 tabular number 风格，方便快速扫描。

第二块是当前批处理流水线，展示 PDF、脚本、确认、TTS、导出五个阶段。每个阶段只显示状态和数量，不展示原始 JSON 或历史日志。批量按钮集中放在流水线下方，包括批量 LangGraph 脚本、批量确认检查、批量音频、合成或导出。

第三块是章节队列。队列按阻塞优先排序，列信息控制在章号、标题、状态、动作四类。每章只显示一个主动作，例如生成脚本、编辑脚本、生成音频、查看。

### 4. 章节编辑展开页

点击“编辑脚本”或“查看”后进入章节展开页，可以是页面内抽屉、侧滑详情页或路由式详情页。第一版优先采用页面内详情模式，减少路由改造风险。

展开页左侧是脚本正文编辑区，按说话人分块展示并支持编辑、锁定和保存。右侧是本章辅助信息，包括审核建议、原文/分析折叠区、本章动作。

审核建议默认只显示需要处理的问题；完整 review report、LangGraph plan、analysis、routing trace、原文片段和历史任务全部放入折叠面板，不进入主屏。

## 组件边界

第一阶段应把当前 `frontend/src/main.tsx` 中的界面逻辑拆成清晰的组件，但不做框架迁移。

建议组件：

- `ProjectSwitcher`：当前项目入口、抽屉、搜索、项目列表、新建和导入入口。
- `BatchOverview`：整本书完成度和阶段统计。
- `BatchPipeline`：五阶段流水线和批量动作。
- `ChapterQueue`：章节队列、状态排序、章节主动作。
- `ChapterScriptDrawer`：脚本编辑、保存确认、审核建议、折叠高级信息。
- `StatusPanel`：待处理数量、错误摘要和低频状态提示。

数据获取和业务状态可以先保留在顶层 App，再通过 props 下传。若拆分后 props 明显过深，再引入局部 reducer；本阶段不新增全局状态库。

## 数据流

页面加载后先获取项目列表和当前项目。若没有当前项目，显示空状态，引导导入 PDF 或新建项目。

选择项目后加载项目详情、章节列表、脚本块、TTS 任务、产物和项目状态。主屏只消费汇总后的生产状态和章节阶段，原始产物数据仅在详情或高级面板里使用。

批量动作沿用现有 API：

- 批量 LangGraph 脚本调用现有批量图接口。
- 单章编辑沿用脚本块保存与确认逻辑。
- TTS、合成、导出沿用现有任务和文件接口。

每次动作完成后刷新项目汇总、章节队列和对应章节详情，避免用户看到过期状态。

## 状态与错误处理

- Loading：主屏使用骨架块，尺寸匹配完成度卡片、流水线和队列表格。
- Empty：无项目时显示导入 PDF 和新建项目入口；无章节时显示解析 PDF 的下一步。
- Error：API 错误以主屏顶部或相关区域内联提示呈现，不使用浏览器弹窗。
- Running：批量任务运行中时，主按钮进入禁用或进行中状态，并保留刷新入口。
- Partial success：批量任务部分失败时，章节队列按阻塞优先显示失败章，并在状态面板列出失败数量。

## 视觉规则

- 色彩保持克制，以浅灰绿背景、白色操作面、深绿主按钮为主，避免高饱和 AI 蓝紫渐变。
- 卡片圆角控制在 8 到 14px；工具按钮和表格行动作保持更小圆角。
- 主屏不使用大段说明文，不做营销式英雄区。
- 章节队列优先表格化和扫描效率，避免每章一张大卡导致滚动负担。
- 交互反馈包括 hover、active、focus-visible 和禁用态。
- 文字不使用视口宽度缩放，长标题允许换行或截断并通过标题属性查看。

## 响应式

桌面端采用顶部项目区、双列概览、章节队列加右侧待处理面板。

窄屏下按以下顺序堆叠：项目区、整本书完成度、当前批处理、待处理摘要、章节队列。项目抽屉宽度不超过视口宽度，章节队列在窄屏切换为紧凑行布局，保留章号、标题、状态、主动作。

## 验证标准

- 用户打开首页后，不展开任何高级区，也能看出整本书总进度和下一步主动作。
- 项目列表入口在主屏可见，点击后能切换项目、搜索项目、导入 PDF 或新建项目。
- 主屏不展示脚本正文、原始 JSON、完整 review report、原文分析和历史日志。
- 点击章节的编辑动作后，可以进入脚本编辑体验，并能找到审核建议和高级信息折叠区。
- 现有 PDF 解析、LangGraph 批量脚本、单章编辑确认、TTS、合成、导出功能仍可访问。
- `npm run build` 通过。
- 浏览器验证覆盖桌面和窄屏，确认主屏信息不重叠、按钮文字不溢出、抽屉可打开和关闭。

## 风险与应对

- 风险：过度隐藏低频功能会让熟悉旧界面的用户找不到入口。
  应对：在章节详情和高级面板保留完整能力，并让项目切换、批量动作、章节主动作始终可见。

- 风险：当前 `main.tsx` 过大，直接大改容易影响业务逻辑。
  应对：先按组件边界做搬迁式拆分，保持数据调用和行为不变，再调整布局。

- 风险：批量任务状态可能来自不同接口，汇总口径不一致。
  应对：实现前先梳理现有状态字段，前端用明确的派生函数生成显示状态，并补充测试或人工检查用例。

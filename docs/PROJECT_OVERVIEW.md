# 项目介绍

## 项目定位

AI 播客生产流水线是一个本地运行的内容生产工具，用于把书籍内容转换为双人对话式播客。它面向个人创作者和小型内容项目，强调可控、可审阅、可分阶段生成，而不是一次性生成不可修改的结果。

## 解决的问题

传统的 AI 播客生成流程容易出现三个问题：

- 生成文本不可控，后期修改困难
- 音频一次性生成，单章出错后重做成本高
- API Key、输入路径、阶段产物散落在脚本和命令中，不方便复用

本项目把流程拆成多个清晰阶段，并提供网页控制台，使用户可以在生成完整音频之前查看和修改每章对话文本。

## 核心体验

用户在网页中创建一个播客项目，填写书籍、目录、评论和 API 配置，然后按阶段运行生产流程。目录会被解析为章节列表，每章都有独立文本和音频状态。用户可以选择任意章节，查看中间文本，修改后保存，并只重做该章音频，最后重新合成完整音频。

## 功能模块

| 模块 | 说明 |
| --- | --- |
| 项目管理 | 保存书籍路径、目录路径、评论路径、LLM 配置、TTS 配置 |
| Phase 1 | 解析书籍和目录，提炼章节知识 |
| Phase 2 | 生成、润色并审核对话脚本 |
| 审阅导出 | 将脚本导出为可人工修改的 Markdown 文本 |
| 章节工作台 | 按目录展示章节，支持折叠、选择、编辑和刷新 |
| 分章音频 | 根据单章结构化对话生成 MP3 |
| 完整合成 | 将所有分章音频合成为完整播客 |
| 任务控制 | 展示进度，支持中止正在运行的制作任务 |

## 技术结构

```text
src/pipeline.js
  负责串联 Phase 1、Phase 2、Review、Audio、Merge

src/agents/
  负责调用不同 Agent 完成评论分析、章节提炼、脚本生成、润色和审核

src/services/
  封装 LLM 和豆包/火山引擎 TTS 调用

src/utils/
  处理 PDF 解析、脚本解析、音频合成、JSON 读写等通用逻辑

src/server.js
  提供本地网页控制台 API

web/
  提供项目输入、阶段控制、章节编辑和音频操作界面
```

## 数据流

```text
输入文件
  -> 章节知识档案
  -> 对话脚本
  -> 可审阅 Markdown
  -> TTS JSON
  -> 分章 MP3
  -> 完整 MP3
```

## 安全边界

项目默认把本地项目数据、运行产物、日志和 `.env` 排除在 Git 之外。网页接口返回项目配置时会隐藏 API Key、Access Token 和 App Key，避免在前端列表中直接暴露密钥。

## 后续可优化方向

- 增加更细粒度的章节状态筛选
- 增加音频试听后的人工标记
- 增加失败任务的重试队列
- 增加配置导入和导出
- 增加多套播客角色模板
